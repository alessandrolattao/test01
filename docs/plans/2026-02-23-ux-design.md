# Recruitment Platform - UX/UI Design Specification

## Theme & Color Scheme

- **DaisyUI Theme**: `corporate`
- **Rationale**: Professional, clean look suitable for a recruitment platform. Neutral tones with clear primary/secondary accents. Good contrast for readability.
- **Configuration** (in `tailwind.config.js` or CSS plugin):
  ```css
  @plugin "daisyui" {
    themes: corporate;
  }
  ```
- **Color usage**:
  - `btn-primary` for main CTAs (submit, next step, save)
  - `btn-secondary` for secondary actions (add question, add answer)
  - `btn-ghost` for tertiary actions (cancel, back)
  - `btn-error` for destructive actions (remove question/answer)
  - `badge-success` for high scores, correct answers
  - `badge-warning` for medium scores
  - `badge-error` for low scores, wrong answers
  - `badge-ghost` for neutral/inactive status

## Global Layout

- **Max container width**: `max-w-4xl mx-auto` for candidate pages, full width with sidebar for admin
- **Base padding**: `p-4 md:p-8`
- **Typography**: Default DaisyUI/Tailwind font stack (system fonts)
- **Responsive breakpoints**: Tailwind defaults (sm:640px, md:768px, lg:1024px, xl:1280px)

**Components**:
- **Steps indicator**:
  ```html
  <ul class="steps steps-horizontal w-full mb-8">
    <li class="step step-primary">Registration</li>
    <li class="step step-primary">Quiz</li>
    <li class="step step-primary">Audio</li>
  </ul>
  ```
- **Recording card**:
  ```html
  <div class="card bg-base-100 shadow-xl">
    <div class="card-body items-center text-center">
      <h2 class="card-title text-2xl mb-2">Audio Presentation</h2>
      <p class="text-base-content/70 mb-6 max-w-md">
        Record a brief presentation about yourself.
        You have up to 2 minutes. Click the button below to start.
      </p>

      <!-- Timer -->
      <div class="text-5xl font-mono font-bold mb-6 tabular-nums">
        <span class="countdown">2:00</span>
      </div>

      <!-- Recording indicator (visible during recording) -->
      <div class="flex items-center gap-2 mb-4">
        <span class="badge badge-error animate-pulse">REC</span>
        <span class="text-sm">Recording in progress...</span>
      </div>

      <!-- Record button -->
      <button class="btn btn-circle btn-error btn-lg w-20 h-20 mb-4">
        <!-- Microphone icon when idle, Stop icon when recording -->
        <svg><!-- mic icon --></svg>
      </button>
      <p class="text-sm text-base-content/50">Click to start recording</p>
    </div>
  </div>
  ```
- **After recording - Playback section** (replaces record button area):
  ```html
  <div class="w-full space-y-4">
    <!-- Native audio player -->
    <audio controls class="w-full"></audio>

    <div class="flex gap-3">
      <button class="btn btn-ghost flex-1">
        Record Again
      </button>
      <button class="btn btn-primary flex-1">
        Submit Recording
      </button>
    </div>
  </div>
  ```

**States**:
1. **Idle**: Mic button visible, timer shows `2:00`, instructions visible
2. **Recording**: Mic button becomes stop button (square icon), timer counts down, `REC` badge pulses, button has `btn-error` style
3. **Recorded**: Audio player appears, "Record Again" + "Submit" buttons shown, timer shows final duration
4. **Submitting**: Submit button shows spinner, "Record Again" is disabled

**Timer behavior**:
- Countdown from `2:00` to `0:00`
- At `0:00`, recording auto-stops
- Timer text turns `text-error` when under 30 seconds
- Uses monospace font (`font-mono tabular-nums`) to prevent layout shifts

**Error handling**:
- Microphone permission denied:
  ```html
  <div class="alert alert-warning">
    <span>Please allow microphone access to record your presentation.</span>
  </div>
  ```
- Upload failure: alert with retry option

**Responsive behavior**:
- Card centered with `max-w-2xl`
- Controls and timer scale appropriately on mobile
- Audio player is full width

**Interactions**:

- "Add Answer" appends a new answer row within the question card
- "Remove" (question): removes entire question card. If only 1 question, button is disabled

- Score field: number input, `min="0"`, integer only

**Validation before save**:
- All questions must have text
- All answers must have text
- Each question must have at least 2 answers
- At least one answer per question must have score > 0
- Show validation errors as `text-error` below the relevant field

**Loading state**:
- Save button shows spinner
- On success: redirect to questionnaire detail page with:
  ```html

    <div class="alert alert-success">
      <span>Questionnaire created successfully</span>
    </div>
  </div>
  ```

## Shared UI Patterns

### Loading States

- **Page-level loading**:
  ```html
  <div class="flex justify-center items-center min-h-[50vh]">
    <span class="loading loading-spinner loading-lg"></span>
  </div>
  ```
- **Button loading**:
  ```html
  <button class="btn btn-primary" disabled>
    <span class="loading loading-spinner loading-sm"></span>
    Saving...
  </button>
  ```

### Error States

- **Inline field error**: `<p class="text-error text-sm mt-1">Error message</p>`
- **Alert banner**:
  ```html
  <div class="alert alert-error mb-4">
    <svg><!-- error icon --></svg>
    <span>Error message here</span>
  </div>
  ```

### Success Feedback

- **Toast notification** (for save/create actions):
  ```html
  <div class="toast toast-end">
    <div class="alert alert-success">
      <span>Action completed successfully</span>
    </div>
  </div>
  ```
  Auto-dismiss after 3 seconds.

### Empty States

- Centered text with `text-base-content/50`, optional CTA button
- Used in: candidates table (no candidates), questionnaires list (no questionnaires)

### Transitions

- Route transitions: none (keep it simple, instant navigation)
- Card/element appear: `transition-opacity duration-200` on dynamic elements (new question cards, answers)
- Hover effects: `hover:bg-base-200 transition-colors` on interactive list items

| Breakpoint | Candidate Pages | Admin Pages |
|---|---|---|
| Mobile (<640px) | Full-width cards, stacked layout, `px-4` padding | Drawer hidden, hamburger menu, table scrolls horizontally |
